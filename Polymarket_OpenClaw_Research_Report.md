## 2. 平台概述

### 2.1 Polymarket 简介

**Polymarket** 是全球最大的去中心化预测市场平台，建立在 Polygon 区块链上。

**核心机制：**
- 用户购买代表事件结果的"Yes"或"No"份额
- 份额价格在 $0.00-$1.00 USDC 之间，代表市场概率
- 正确结果的份额最终支付 $1.00
- 交易对手是其他用户，而非"庄家"

**关键特性：**
- **价格 = 概率**：YES 份额价格 0.63 表示 63% 的市场预期概率
- **可随时卖出**：无需持有到结算
- **无账户限制**：赢家不会被封禁
- **UMA 乐观预言机**：去中心化事件结算

**2025年里程碑：**
- 获得 ICE（纽交所母公司）$20亿投资
- 估值约 $90亿
- CFTC 批准重返美国市场

### 2.2 OpenClaw 简介

**OpenClaw** 是一个开源 AI 代理框架，专为构建自主代理而设计。

**核心能力：**
- **持久记忆** - 维护跨会话的上下文和知识
- **工具调用编排** - 连接外部 API 和服务
- **技能系统** - 可组合的模块化能力扩展
- **24/7 自主运行** - 无需人工干预

**技术架构：**
```
OpenClaw Agent
├── Memory Layer (持久记忆)
├── Skill Registry (技能注册表)
├── Tool Calling (工具调用引擎)
└── Execution Environment (执行环境)
```

**关键技能库 - BankrBot：**
- Polymarket 交易技能
- 交易所连接器（Binance、Coinbase、Kraken）
- DeFi 交互技能
- 分析工具

### 2.3 PolyClaw 简介

**PolyClaw** 是首个专为 AI 代理构建的预测市场平台。

**核心差异：**
| 特性 | Polymarket | PolyClaw |
|------|-----------|----------|
| 目标用户 | 人类 | AI 代理 |
| 钱包集成 | 手动连接 | x402 原生（出生即有） |
| 支付方式 | API 密钥/订阅 | 按请求付费 |
| 运行模式 | 人机交互 | 全自主 24/7 |

**工作流程：**
1. **SPAWN（生成）** - 选择模板或自定义创建代理
2. **FUND（注资）** - 存入 ETH，设置预算
3. **TRADE（交易）** - 代理自主进入市场交易
4. **EARN（盈利）** - 赢家获得 $PolyClaw 代币

## 4. 集成策略与最佳实践

### 4.1 自动化做市商策略

**核心原理：**
- 在买卖两侧持续挂单
- 赚取买卖价差（spread）
- 承担库存风险

**OpenClaw 做市商架构：**
```
Market Analysis Skill    → 扫描流动性缺口和错误定价
        ↓
Execution Engine Skill   → 毫秒级跨市场下单
        ↓
Risk Management Skill    → 监控敞口，设置仓位限制
```

**关键参数配置：**
```python
# 做市商配置示例
config = {
    "spread_target": 0.02,        # 目标价差 2%
    "position_limit": 10000,      # 单市场仓位上限 $10,000
    "order_size": 500,            # 单笔订单大小
    "rebalance_threshold": 0.3,   # 再平衡触发阈值
    "markets_count": 31,          # 同时覆盖市场数
}
```

**收益记录：**
- 某机器人 7 天内执行 47,000+ 笔交易
- 覆盖 31 个市场
- 净利润 $115,000（扣除费用后）
- 平均每笔利润约 $2.45

### 4.2 套利交易策略

**套利类型：**

#### A. 单市场套利
当 YES + NO 价格之和 < $1.00 时：
```
示例：YES = $0.48, NO = $0.50
总和 = $0.98
套利空间 = $0.02/份额
```
- 2024-2025年已产生超过 $3950万利润

#### B. 跨平台套利
Polymarket vs Kalshi 价格差异：

| 市场类型 | 平均套利空间 | 风险等级 |
|---------|-------------|---------|
| 政治市场 | 1-5% | 中 |
| 体育市场 | 0.5-2% | 低 |
| 加密市场 | 2-10% | 高 |

**重要警告：** 结算机制不同可能导致同事件不同结果
- 2024年政府停摆市场：Polymarket 结算"Yes"，Kalshi 结算"No"

#### C. 时间套利
利用 Polymarket 价格滞后于交易所现货：
- BTC 15分钟市场利润记录：$313 → $414,000/月
- 延迟窗口：通常 100-500 毫秒

**OpenClaw 套利机器人配置：**
```
ArbitrageBot:
  skills:
    - market_scanner      # 扫描多平台价格
    - opportunity_finder  # 识别套利机会
    - execution_engine    # 快速执行
    - position_tracker    # 仓位追踪

  config:
    min_profit_threshold: 0.005  # 最低 0.5% 利润
    max_position_size: 5000      # 最大仓位
    slippage_tolerance: 0.001    # 滑点容忍度
```

## 6. 风险管理

### 6.1 市场风险

| 风险类型 | 描述 | 缓解措施 |
|---------|------|---------|
| 波动性风险 | 新闻事件导致剧烈波动 | 设置止损，限制单市场敞口 |
| 流动性风险 | 薄市场滑点巨大 | 避免市价单，使用限价单 |
| 相关性风险 | 相关市场联动 | 跨市场对冲 |
| 结算风险 | UMA 预言机争议 | 理解结算机制，参与治理 |

### 6.2 技术风险

| 风险类型 | 描述 | 缓解措施 |
|---------|------|---------|
| 智能合约风险 | 代码漏洞 | 只投入可承受损失的资金 |
| API 故障 | 连接中断 | 多节点部署，备用连接 |
| 延迟问题 | 执行延迟 | 使用 VPS，低延迟网络 |
| 技能供应链 | 恶意技能 | 只使用已验证发布者的技能 |

### 6.3 监管风险

| 风险类型 | 描述 | 缓解措施 |
|---------|------|---------|
| 司法管辖区限制 | 部分地区禁止 | 了解当地法规 |
| 平台冻结 | 执法行动 | 分散资金，使用多平台 |
| 税务合规 | 收益申报 | 保留完整记录 |

### 6.4 资金管理原则

```
1. 仓位限制
   ├── 单市场最大敞口：总资金 10%
   ├── 单策略最大分配：总资金 30%
   └── 保留现金：至少 20%

2. 止损设置
   ├── 单笔交易止损：-5%
   ├── 日止损：-10%
   └── 周止损：-20%（触发后暂停策略）

3. 分散配置
   ├── 多市场覆盖
   ├── 多策略组合
   └── 多平台部署
```

## 7. 安全最佳实践

### 7.1 恶意技能威胁

**Snyk 发现（2026年初）：**
- 发现 386 个恶意技能发布到 BankrBot 注册表
- 攻击向量包括：
  - 凭据窃取（API 密钥、私钥）
  - 交易重定向（小额抽成）
  - 钱包 draining（授权后转账）
  - 依赖污染（引入恶意子依赖）

### 7.2 安全检查清单

```
□ 技能验证
  ├── 只使用已验证发布者的技能
  ├── 审计源代码（特别注意网络调用）
  └── 检查依赖项

□ 钱包安全
  ├── 使用专用沙盒钱包
  ├── 永不连接主钱包
  └── 限制资金额度

□ 权限最小化
  ├── 只授予必要权限
  ├── 市场数据技能不需要提款权限
  └── 定期审计已安装技能

□ 监控与警报
  ├── 独立监控所有钱包活动
  ├── 设置异常交易警报
  └── 定期审计交易历史

□ 密钥管理
  ├── 永不硬编码 API 密钥或私钥
  ├── 使用环境变量或密钥管理工具
  └── 在 VPS 上使用安全存储
```

## 9. 结论与建议

### 9.1 最佳用法总结

**推荐策略优先级：**

```
1. 做市商策略（推荐指数：★★★★★）
   - 适合：有编程基础，资金量中等
   - 收益：稳定，依赖于规模
   - 风险：中等（库存风险）

2. 跨平台套利（推荐指数：★★★★☆）
   - 适合：技术能力强，快速执行
   - 收益：机会性，需要监控
   - 风险：结算机制差异

3. 跟单交易（推荐指数：★★★☆☆）
   - 适合：新手，资金量小
   - 收益：跟随成功交易者
   - 风险：依赖他人策略

4. 高频交易（推荐指数：★★☆☆☆）
   - 适合：专业团队，大资金
   - 收益：高但竞争激烈
   - 风险：高（技术、监管）
```

### 9.2 入门建议

**新手路径：**
